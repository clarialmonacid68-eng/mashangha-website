## 已验证事实

1. 工信部备案管理系统为 `https://beian.miit.gov.cn/`。如果选择中国大陆服务器，应通过接入服务商完成网站备案，并在网站页脚展示备案号。
2. Supabase Phone Login 需要另行配置短信服务商。其托管文档列出的服务商为 MessageBird、Twilio、Vonage 和 TextLocal，不能据此假设中国大陆验证码已可用。
3. 腾讯云国内短信需要账号实名、短信实名资质、签名和正文模板。签名通过云平台审核后还需要运营商实名报备。
4. Vercel 可以配置函数执行区域，但生产可用性仍必须通过目标用户网络实测。
5. 微信支付的业务准入、二级商户、退款和分账不能只根据通用 API 文档推断，必须取得当前商户主体下的可追溯确认。
6. `https://www.mshcode.com/` 已上线。2026-06-15 检查返回 HTTP 200，响应头包含 `server: Vercel` 和 `x-vercel-id: sin1`。
7. 从当前网络进行的 10 次短时采样全部成功，TTFB 最低约 0.288 秒、最高约 0.422 秒。该结果只作为基线，不能替代移动、联通、电信三网测试。
8. 用户提供的阿里云域名列表截图显示 `mshcode.com`、`mshcode.cn`、`mshcode.net` 和 `mshcode.com.cn` 均为正常状态；`mshcode.com` 注册时间为 2026-06-04，到期时间为 2027-06-04。
9. 域名注册、域名实名认证和 ICP 备案是三件不同事项。当前证据可确认域名持有，不能确认 ICP 备案。

## 必须完成的实测

### 部署测试

对以下候选部署同一个最小应用：

| 候选 | 状态 | 测试内容 |
|---|---|---|
| Vercel + Supabase | 未测试 | 首页 TTFB、API 延迟、登录、上传 |
| 中国香港/新加坡 Node + PostgreSQL | 未测试 | 首页 TTFB、API 延迟、登录、上传 |
| 中国大陆云 + PostgreSQL + 对象存储 | 未测试 | 备案完成后测试 |

每 5 分钟执行一次探测，持续 24 小时。记录中国移动、中国联通、中国电信网络下的：

- 成功率
- P50/P95 TTFB
- API P50/P95
- 1 MB 和 20 MB 文件上传成功率

通过标准：

```text
页面和 API 成功率 >= 99.5%
首页 P95 TTFB <= 1.5 秒
核心 API P95 <= 2 秒
20 MB 上传成功率 >= 99%
```

### OTP 测试

- 手机：移动、联通、电信各至少 1 个号码。
- 邮箱：QQ、163、企业邮箱各至少 1 个地址。
- 每个目标至少发送 20 次，分布在工作日白天和晚上。

通过标准：

```text
成功到达率 >= 95%
到达时间 P95 <= 60 秒
所有失败都有服务商错误码或邮件退信记录
```

## 架构闸门

进入 Task 1 前至少需要确定：

- 开发环境可暂用 Supabase。
- 生产部署候选和迁移边界已明确。
- 国内短信申请已启动，邮箱 SMTP 候选已选定。
- 第一阶段明确不开放真实付款。

进入 Task 16 前必须全部满足：

- 微信支付业务类目已批准。
- 二级商户进件方案已确认。
- 支付、退款、分账和账单权限已开通。
- 资金流和平台收费规则已经法务、财务确认。
