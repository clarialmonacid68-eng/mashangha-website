## 申请目标

码上好计划申请微信支付服务商/平台型能力，用于支持客户向平台订单全额付款，开发者作为二级商户承接服务，订单验收后执行一次分账。平台收取约定技术服务佣金，不提供充值、余额、提现或资金池能力。

- 运营主体营业执照、统一社会信用代码、注册地址和经营范围。
- 法定代表人身份证明、联系方式和授权材料。
- 对公银行账户信息。
- 平台联系人、客服电话、投诉邮箱和紧急联系人。
- 域名 `mshcode.com`、`mshcode.cn`、`mshcode.net`、`mshcode.com.cn` 的持有人证明。
- ICP 备案截图或备案号；未完成备案前，生产支付申请保持 blocked。
- 网站可访问截图：首页、需求页、开发者页、服务协议、隐私政策、交易规则、退款规则、投诉入口。

## 平台规则材料

- 平台服务协议：说明客户、平台、开发者的权利义务。
- 隐私政策：说明账号、实名、联系方式、订单、附件和支付相关数据的收集、使用、保存和删除方式。
- 交易规则：说明需求发布、审核、报价、选标、全额付款、履约、交付、验收、评价和争议流程。
- 退款规则：首版只支持未完成结算订单的全额退款；已分账退款需微信支付和财务规则确认后再开放。
- 投诉规则：说明投诉入口、处理时限、证据要求、人工仲裁结论和申诉方式。
- 开发者入驻规则：说明主体要求、实名审核、作品审核、服务范围、收款主体和违规处理。

## 业务关系说明

```text
客户：发布软件开发需求，选择开发者报价后一次性全额付款。
平台：审核需求和开发者，撮合报价，保存订单证据，处理投诉和争议，按规则收取技术服务佣金。
开发者：作为二级商户或经确认允许的服务提供方，完成软件开发交付，订单验收后获得分账款项。
微信支付：提供支付、退款、分账、账单和二级商户能力，具体权限以后台/合同确认结果为准。
```

## 二级商户入驻流程草案

1. 开发者提交入驻申请，平台只收集必要业务资料和联系方式。
2. 平台审核作品、服务范围、履约能力和违规风险。
3. 涉及身份证件、银行卡等高敏资料时，优先直接提交微信支付接口；平台数据库只保存渠道申请号、状态、脱敏摘要和审核记录。
4. 入驻状态固定为 `not_started -> submitting -> reviewing -> approved/rejected`。
5. 只有 `approved` 的开发者可以进入真实支付订单。
