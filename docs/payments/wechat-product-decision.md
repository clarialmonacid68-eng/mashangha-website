# 微信支付产品决策

**状态：** blocked  
**日期：** 2026-06-15  
**产品方向：** 平台型支付、开发者二级商户、验收后单次分账

## 已确认的内部业务模型

- 客户选择报价后一次性全额付款。
- 每个订单只有一次正式验收和一次分账。
- 平台只向开发者收取成交佣金。
- 不建立平台余额、充值或用户自行提现。
- 首版退款只支持全额退款。
- 退款、分账和渠道费用独立记账。

## 给微信支付支持的业务描述

```text
码上好是软件开发服务撮合平台。客户发布开发需求，平台审核后由认证开发者报价。
客户选择报价并一次性全额付款。开发者完成履约并提交交付物，客户验收后，
支付机构向开发者二级商户进行单次分账，平台保留约定技术服务佣金。
平台不提供用户余额、充值或自行提现，不自行沉淀或人工转付客户资金。
发生争议时平台根据需求、报价、留言、交付和验收记录进行人工仲裁；
首版仅支持未完成结算订单的全额退款。
```
