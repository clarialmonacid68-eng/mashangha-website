# 码上好正式运营启动清单

## 当前结论

网站现在适合作为“试运营获客页”，但还不能作为完整交易平台运营。

已具备：

- 自定义域名与 Vercel 静态部署
- 首页、需求展示、开发者展示、需求登记、开发者入驻页面
- 桌面端和移动端基础响应式布局
- 搜索引擎描述、`robots.txt`、`sitemap.xml` 和 404 页面
- 试运营与示例数据标识

尚未具备：

- 真实表单收件和后台线索管理
- 登录、注册和用户身份系统
- 真实需求、开发者和项目数据库
- 消息、报价、订单和交付流程
- 支付、退款、发票或资金托管能力
- 正式服务协议、隐私政策和运营主体信息
- 数据分析、错误监控和转化统计

## 上线前必须完成

### P0：先让线索能够被收到

1. 确定需求提交后的收件渠道：
   - 企业微信或个人微信
   - 企业邮箱
   - 飞书多维表格、腾讯文档或独立数据库
2. 确定开发者入驻申请的收件渠道。
3. 给两个表单接入真实提交接口，并增加：
   - 必填校验
   - 联系方式格式校验
   - 用户同意隐私说明
   - 成功和失败反馈
   - 防垃圾提交
4. 每天至少检查两次新线索，并建立跟进状态：
   - 新提交
   - 已联系
   - 需求已确认
   - 匹配中
   - 已成交
   - 暂不跟进

在 P0 完成前，不建议购买广告流量。

### P1：建立可信的首批供给

1. 人工招募 5-10 位真实开发者。
2. 核验身份、作品链接、技能、可接单状态和价格范围。
3. 获得本人同意后再公开展示资料。
4. 删除所有示例人物，换成真实且可核验的数据。
5. 先采用人工匹配，不急着开发复杂的自动报价系统。

### P1：获得首批真实需求

1. 选择一个清晰的早期客群，例如：
   - 本地商家小程序
   - 企业内部管理工具
   - AI 客服与知识库
   - 自动化工作流
2. 为该客群准备 3 个标准服务包：
   - 交付内容
   - 大致价格区间
   - 预计周期
   - 不包含的范围
3. 通过熟人、行业微信群、内容平台和线下商家获得前 10 条需求。
4. 每条需求都记录来源、预算、成交原因和流失原因。

### P1：补齐信任与合规页面

正式收集联系方式之前，至少准备：

- 运营主体或负责人信息
- 联系方式
- 隐私说明
- 服务规则
- 投诉和纠纷处理方式
- 数据删除或撤回渠道

在没有取得相应能力前，不要宣传“平台资金托管”“担保交易”“保证退款”等服务。

## 推荐的最小技术方案

第一阶段无需立刻重构 Next.js，可以保留静态页面：

- 前端：现有 HTML/CSS
- 表单：Vercel Serverless Function
- 数据库：Supabase PostgreSQL
- 通知：企业邮箱或企业微信机器人
- 统计：Vercel Web Analytics 或其他合规统计工具
- 线索后台：Supabase Dashboard，后续再开发管理后台

建议先建两张表：

### `demand_leads`

- `id`
- `title`
- `project_type`
- `description`
- `budget`
- `delivery_time`
- `cooperation_mode`
- `contact_name`
- `contact_value`
- `status`
- `source`
- `created_at`

### `developer_applications`

- `id`
- `name`
- `contact_value`
- `city`
- `specialty`
- `tools`
- `starting_price`
- `portfolio_url`
- `status`
- `created_at`

## 30 天运营节奏

### 第 1 周

- 接通两个表单
- 补齐隐私说明和联系方式
- 招募 5 位真实开发者
- 确定一个主要客户群

### 第 2 周

- 获取 10 条真实需求
- 完成人工需求访谈
- 建立开发者匹配表和跟进表

### 第 3 周

- 推进 2-3 个真实报价
- 记录常见需求、预算异议和流失原因
- 根据真实反馈调整首页文案

### 第 4 周

- 争取完成首个付费项目
- 取得可公开的案例与评价授权
- 决定是否投入用户系统、订单系统和支付能力

## 需要运营者提供的资料

要继续完成 P0，需明确：

1. 需求线索发送到哪里。
2. 开发者申请发送到哪里。
3. 网站公开显示的联系人、微信或邮箱。
4. 运营主体名称；若暂未注册公司，明确当前使用个人还是筹备中主体。
5. 是否已有 Supabase、企业邮箱或企业微信。
